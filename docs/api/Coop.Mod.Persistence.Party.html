MobilePartySync

Synchronization implementation for CampaignMapMovement. It handles buffered changes to 2 different field groups:

  1. The captured MovementData.
  2. The captured 2d position on the campaign map. These changes are forwarded to the responsible IMovementHandler. The following handlers are registered:
  3. On every client: MobilePartyEntityClient for every party that is directly controlled by this local client, i.e. its main party.
  4. On the server: MobilePartyEntityServer for every party that is not directly controlled by any client.
